# @trunkjs/demo-viewer-frontend

## Purpose

Browser-side Laufzeit für den Demo-Viewer:
- Web Components
- `DemoRegistry`
- Demo-Typen und `defineDemo(...)`

Das Vite-Plugin liegt **nicht** hier, sondern in `@trunkjs/vite-demo-viewer`.

## Import

```ts
import '@trunkjs/demo-viewer-frontend';
```

Oder mit einzelnen Exports:

```ts
import { DemoRegistry, defineDemo } from '@trunkjs/demo-viewer-frontend';
```

## Beispiele

### Viewer registrieren

```ts
import '@trunkjs/demo-viewer-frontend';

const viewer = document.createElement('tj-demo-viewer');
viewer.demos = [];
document.body.append(viewer);
```

### Demo definieren

```ts
import { defineDemo } from '@trunkjs/demo-viewer-frontend';

export default defineDemo({
  title: 'Hallo',
  html: '<p>Demo</p>',
});
```

### Navigation aus Demos ableiten

```ts
import { DemoRegistry } from '@trunkjs/demo-viewer-frontend';

const registry = new DemoRegistry([
  { filename: 'forms/button.demo.ts', title: 'Button' },
]);

const nav = registry.getNavData();
```

## Dev mockup

Für die lokale Entwicklung gibt es ein kleines Standalone-Mockup mit denselben zentralen Dev-Server-Dateien/-Routen wie beim Plugin:

- `index.html`
- `src/mock/tjDemoViewerClient.js`
- `virtual:tdemo-registry`
- Route `/__tdemo`
- Beispiel-Demos unter `demo/**/*.demo.ts`

## Agent hints

- Dieses Paket enthält die Browser-/Frontend-Seite des Demo-Viewers.
- Das Vite-Plugin bleibt in `@trunkjs/vite-demo-viewer`.
- Für Browser-UI, Registry und Viewer-Typen dieses Paket verwenden.
- Das Paket kann lokal standalone über den Vite-Dev-Server geöffnet werden; dafür dient das Mockup in `index.html` + `src/mock/tjDemoViewerClient.js`.
- `<tj-demo-renderer>` verarbeitet `html`, `markdown`, `wrapper_html`, `css` und `render(root)`.
- Bei `css === undefined` oder `'default'` wird automatisch das Default-Stylesheet injiziert.
