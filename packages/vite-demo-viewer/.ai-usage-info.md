# @trunkjs/vite-demo-viewer

## Purpose

Vite-Plugin zum Auffinden und Anzeigen von Demo-Dateien in einem einfachen Demo-Viewer.

Zusätzlich gibt es mit `viteDemoExporter` einen statischen Exporter für `index.html`, `index.js` und gebündelte Assets.

## Wichtig: Dateinamen der Demos

Demo-Dateien müssen auf **`.demo.ts`** enden.

Beispiele:

```text
button.demo.ts
form.demo.ts
markdown.demo.ts
```

Standardmäßig werden nur `**/*.demo.ts` gescannt.

Die ältere Endung `.tdemo.ts` soll nicht mehr verwendet werden und wird standardmäßig nicht mehr berücksichtigt.

## Import

```ts
import { defineDemo, tjDemoViewerPlugin, viteDemoExporter } from '@trunkjs/vite-demo-viewer';
```

## Vite setup

Minimal:

```ts
import { defineConfig } from 'vite';
import { tjDemoViewerPlugin } from '@trunkjs/vite-demo-viewer';

export default defineConfig({
  plugins: [tjDemoViewerPlugin()],
});
```

Mit Optionen:

```ts
import { defineConfig } from 'vite';
import { tjDemoViewerPlugin } from '@trunkjs/vite-demo-viewer';

export default defineConfig({
  plugins: [
    tjDemoViewerPlugin({
      include: ['packages/ui/demo/**/*.demo.ts'],
      route: '/__tdemo',
    }),
  ],
});
```

## Plugin options

- `include?: string[]`
  - Demo-Glob-Patterns
  - Default: `['**/*.demo.ts']`
- `route?: string`
  - Dev-Route des Viewers
  - Default: `'/__tdemo'`

## Static export

```ts
import { viteDemoExporter } from '@trunkjs/vite-demo-viewer';

await new viteDemoExporter('dist/demo-static').build();
```

Mit Optionen:

```ts
await new viteDemoExporter('dist/demo-static', {
  include: ['packages/ui/demo/**/*.demo.ts'],
  title: 'UI Demo Viewer',
}).build();
```

## Exporter options

- `outDir: string`
  - Zielverzeichnis
- `include?: string[]`
  - Demo-Glob-Patterns
  - Default: `['**/*.demo.ts']`
- `title?: string`
  - HTML-Title
  - Default: `'TDemo Viewer'`

## Demos definieren

### Als Objekt

```ts
import { defineDemo } from '@trunkjs/vite-demo-viewer';

export default defineDemo({
  title: 'Meine Demo',
  description: 'Kurze Beschreibung',
  html: '<p>Hallo Demo</p>',
});
```

### Als render-Demo

```ts
export function render(root: HTMLElement) {
  root.textContent = 'Hallo Demo';
}
```

Oder:

```ts
export default {
  title: 'Counter',
  render(root: HTMLElement) {
    root.textContent = 'Counter Demo';
  },
};
```

## Wichtige Demo-Optionen (`TDemoDefinition`)

- `filename?: string`
  - wird normalerweise automatisch gesetzt
- `group?: string`
- `tags?: string[]`
- `title?: string`
- `description?: string`
- `html?: string`
- `markdown?: string`
- `wrapper_html?: string`
  - nutzt `{{content}}` als Platzhalter
- `css?: string | 'default' | null | Array<string | 'default'>`
- `controls_raw_html?: string`
- `controls?: TControlDefinition[]`
- `render?(root: HTMLElement)`

## CSS behavior

- `css === undefined`
  - Default-Style wird automatisch injiziert
- `css === 'default'`
  - Default-Style explizit aktivieren
- `css === null`
  - keine automatische CSS-Injektion
- `css === string`
  - Inline-CSS oder Stylesheet-URL
- `css === ['default', customCss]`
  - Default-Style plus eigene Styles

Der Renderer nutzt dafür `src/components/tj-demo-renderer/default-style.scss`, das intern `src/style/codestyle.scss` einbindet.

## Beispiele

### HTML mit externer SCSS-Datei

```ts
import { defineDemo } from '@trunkjs/vite-demo-viewer';
import html from './demo.html?raw';
import styleUrl from './demo.scss?url';

export default defineDemo({
  title: 'HTML + SCSS',
  html,
  css: styleUrl,
});
```

### HTML mit inline-SCSS

```ts
import { defineDemo } from '@trunkjs/vite-demo-viewer';
import style from './demo.scss?inline';

export default defineDemo({
  title: 'Inline SCSS',
  css: ['default', style],
  html: '<div class="box">Hallo</div>',
});
```

### Markdown-Demo

```ts
import { defineDemo } from '@trunkjs/vite-demo-viewer';

export default defineDemo({
  title: 'Markdown Demo',
  markdown: `# Hallo

- Punkt 1
- Punkt 2`,
});
```

### Markdown mit Wrapper

```ts
import { defineDemo } from '@trunkjs/vite-demo-viewer';
import style from './article.scss?inline';

export default defineDemo({
  title: 'Markdown im Wrapper',
  css: ['default', style],
  wrapper_html: '<article class="article">{{content}}</article>',
  markdown: '## Inhalt',
});
```

### Interaktive Demo

```ts
export default {
  title: 'Counter',
  render(root: HTMLElement) {
    let count = 0;

    const value = document.createElement('output');
    const button = document.createElement('button');
    button.textContent = '+1';

    const update = () => {
      value.textContent = String(count);
    };

    button.addEventListener('click', () => {
      count += 1;
      update();
    });

    update();
    root.append(value, button);
  },
};
```

### Controls

```ts
import { defineDemo } from '@trunkjs/vite-demo-viewer';

export default defineDemo({
  title: 'Demo mit Controls',
  html: '<p>Öffne den Controls-Bereich unten.</p>',
  controls: [
    {
      label: 'Klick mich',
      element: 'button',
      onclick: () => console.log('geklickt'),
    },
    {
      label: 'Auswahl',
      element: 'select',
      selectOptions: ['A', 'B', 'C'],
      onchange: (event) => console.log((event.target as HTMLSelectElement).value),
    },
  ],
});
```

### Eigene Controls als HTML

```ts
import { defineDemo } from '@trunkjs/vite-demo-viewer';

export default defineDemo({
  title: 'Custom Controls',
  html: '<p>Mit eigenem Controls-Markup</p>',
  controls_raw_html: `
    <button onclick="console.log('custom')">Custom Button</button>
  `,
});
```

## Agent hints

- Demo-Dateien müssen auf `.demo.ts` enden.
- Standardmäßig werden `**/*.demo.ts` gescannt.
- `viteDemoExporter` erzeugt einen statisch deploybaren Viewer-Build mit `index.html`, `index.js` und gebündelten Assets im angegebenen `outDir`.
- Demos können `render(root)` als named Export oder im `default`-Export anbieten.
- Für Workspace-Demos ggf. das `include`-Pattern explizit setzen.
- Der Viewer wird über `<tj-demo-viewer>` gerendert und nutzt intern `<tj-demo-viewer-nav>`.
- `DemoRegistry` kapselt die Business-Logik für Demo-Listen, Navigation und Lookup per Hash/Filename.
- Einzelne Demo-Flächen können mit `<tj-demo>` aufgebaut werden.
- `tj-demo` bietet einen schwarzen Header mit `title` und `description`, einen Default-Slot für den Inhalt, einen `header`-Slot und integriert `<tj-demo-controls>` für den fixierten Controls-Bereich am unteren Rand.
- `<tj-demo-controls>` bekommt eingebaute Controls über `data` und rendert diese im Shadow DOM; zusätzliche Controls können über den `controls`-Slot eingebunden werden.
- `<tj-demo-renderer>` ist ein ungestylter Renderer mit nur einem Slot und einer imperativen `showDemo(demo)`-Methode.
- Im Renderer gibt es `src/components/tj-demo-renderer/default-style.scss`; dieses bindet das Mixin `src/style/codestyle.scss` ein und wird automatisch verwendet, wenn `demo.css` `undefined` oder `'default'` ist. Mit `css: null` kann die automatische Default-Formatierung deaktiviert werden.
- `demo.markdown` wird im Renderer über `@trunkjs/ast-markdown` (`MarkdownDocument`) in DOM umgewandelt; bei `wrapper_html` wird der gerenderte Markdown-HTML-Output in `{{content}}` eingesetzt.
- Für statische Content-/Markdown-Flächen gibt es `src/style/codestyle.scss` mit dem Mixin `codestyle()`, das eine neutrale Basisformatierung für Headings, Text, Links, Listen, Blockquotes, Tabellen sowie `pre`/`code` liefert.
- `TDemoDefinition.css` unterstützt Inline-CSS (`<style>`) und Stylesheet-Referenzen (`<link rel="stylesheet">`). Dadurch funktionieren sowohl `style.scss?inline` als auch importierte Stylesheet-Pfade/URLs.
- Navigationsdaten folgen `TNavData` mit `title`, `description` und einem `tree` aus Branch-/Leaf-Nodes.
- Aufgeklappte Navigationsknoten werden im `sessionStorage` gespeichert.
