# @trunkjs/vite-demo-viewer

## Purpose

Vite-Plugin zum Auffinden und Anzeigen von `*.tdemo.ts`-Dateien in einem einfachen Demo-Viewer.

## Import

```ts
import { tjDemoViewerPlugin } from '@trunkjs/vite-demo-viewer';
```

## Example

```ts
import { defineConfig } from 'vite';
import { tjDemoViewerPlugin } from '@trunkjs/vite-demo-viewer';

export default defineConfig({
  plugins: [tjDemoViewerPlugin()],
});
```

Eine Demo-Datei kann z. B. so aussehen:

```ts
export function render(root: HTMLElement) {
  root.textContent = 'Hallo Demo';
}
```

## Agent hints

- Standardmäßig werden `**/*.tdemo.ts` gescannt.
- Demos können `render(root)` als named Export oder im `default`-Export anbieten.
- Für Workspace-Demos ggf. das `include`-Pattern explizit setzen.
- Der Viewer wird über `<tj-demo-viewer>` gerendert und nutzt intern `<tj-demo-viewer-nav>`.
- `DemoRegistry` kapselt die Business-Logik für Demo-Listen, Navigation und Lookup per Hash/Filename.
- Einzelne Demo-Flächen können mit `<tj-demo>` aufgebaut werden.
- `tj-demo` bietet einen schwarzen Header mit `title` und `description`, einen Default-Slot für den Inhalt, einen `header`-Slot und integriert `<tj-demo-controls>` für den fixierten Controls-Bereich am unteren Rand.
- `<tj-demo-controls>` bekommt eingebaute Controls über `data` und rendert diese im Shadow DOM; zusätzliche Controls können über den `controls`-Slot eingebunden werden.
- Navigationsdaten folgen `TNavData` mit `title`, `description` und einem `tree` aus Branch-/Leaf-Nodes.
- Aufgeklappte Navigationsknoten werden im `sessionStorage` gespeichert.

## Navigation example

```ts
const navData = {
  title: 'TDemos',
  description: 'Gefundene Demo-Dateien',
  tree: [
    {
      name: 'forms',
      children: [
        { name: 'input-basic', href: '#/demo/forms%2Finput-basic.tdemo.ts' },
      ],
    },
  ],
};
```
