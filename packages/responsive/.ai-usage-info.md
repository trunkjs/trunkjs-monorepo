# @trunkjs/responsive

## Purpose

Responsive DOM behavior without writing custom resize logic.

Use it when elements inside a wrapper should change:
- classes
- inline styles
- visibility
- layout utility classes

Everything only works inside `<tj-responsive>`.

## Import

```ts
import '@trunkjs/responsive';
```

## Main usage

```html
<tj-responsive>
  <div class="-sm:d-none sm:d-block">Visible from `sm` and up</div>
</tj-responsive>
```

## Class syntax examples

### Up to a breakpoint

```html
<div class="-md:d-none">hidden on xs/sm/md</div>
```

### From a breakpoint and up

```html
<div class="lg:d-flex">flex on lg/xl/xxl</div>
```

### Between breakpoints

```html
<div class="md-xl:d-block">visible from md up to before xl</div>
```

### Multiple classes at one breakpoint

```html
<div class="lg:card.shadow-lg.border"></div>
```

### Chain syntax

```html
<div class="d-none:md:d-block:xl:d-flex"></div>
```

Meaning:
- before `md` => `d-none`
- `md` to before `xl` => `d-block`
- `xl` and up => `d-flex`

## Responsive inline styles

Use `style-{breakpoint}` attributes.

```html
<tj-responsive>
  <div
    style="display:none;color:black"
    style-sm="display:block"
    style-lg="display:flex;color:red"
  ></div>
</tj-responsive>
```

Behavior:
- base style stays the fallback
- matching breakpoint styles override observed properties
- later breakpoints win

## Typical examples

### Mobile-only / desktop-only

```html
<tj-responsive>
  <div class="-md:d-block md:d-none">mobile only</div>
  <div class="-md:d-none md:d-block">desktop and up</div>
</tj-responsive>
```

### Switch layout direction

```html
<tj-responsive>
  <section class="d-flex -md:flex-column md:flex-row"></section>
</tj-responsive>
```

### Change style by breakpoint

```html
<tj-responsive>
  <aside style="max-width:100%" style-md="max-width:50%" style-xl="max-width:33%"></aside>
</tj-responsive>
```

## Breakpoints

Default names:

```ts
xs sm md lg xl xxl
```

Default widths come from `@trunkjs/browser-utils`:
- `xs`: 0
- `sm`: 576
- `md`: 768
- `lg`: 992
- `xl`: 1200
- `xxl`: 1400

## Behavior notes

- Only descendants of `<tj-responsive>` are processed.
- DOM changes inside the wrapper are observed automatically.
- On breakpoint change, matching classes/styles are recalculated.
- Add `debug` on `<tj-responsive>` for logs.

## Debug example

```html
<tj-responsive debug>
  <div class="-sm:d-none sm:d-block"></div>
</tj-responsive>
```

## Agent hints

- Prefer this package over handwritten `resize` listeners for simple responsive DOM updates.
- Do not create media classes or media queries in CSS used only in single HTML Files. 
- Put `<tj-responsive>` near body element. It will observe the complete tree.
- Use class syntax for utility-class toggling.
- Use `style-{bp}` only when class toggling is not enough.
